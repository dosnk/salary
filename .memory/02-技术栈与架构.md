### 后端核心 API 路由

| 路由前缀 | 用途 |
|---------|-----|
| `POST /v1/auth/login` | 登录 |
| `POST /v1/auth/refresh` | 刷新 Token（关键！Android 端拦截器调用） |
| `POST /v1/auth/logout` | 登出 |
| `GET /v1/projects` | 工程列表（支持 year-month 过滤、分页） |
| `POST /v1/projects` | 创建工程 |
| `GET /v1/projects/:id` | 工程详情 |
| `PUT /v1/projects/:id` | 修改工程 |
| `PUT /v1/projects/:id/status` | 变更工程状态 |
| `POST /v1/projects/:id/subprojects` | 添加子项目 |
| `POST /v1/settlements/calculate` | **计算结算金额（唯一入口，前端禁用本地计算）** |
| `POST /v1/settlements` | 提交结算 |
| `GET /v1/statistics/dashboard` | 统计概览 |
| `GET /v1/statistics/monthly/:yearMonth` | 月度统计 |
| `GET /v1/ai/config` / `PUT /v1/ai/config` | AI 配置（仅 admin） |
| `POST /v1/ai/chat` | AI 对话（SSE 流式） |

### Android 关键交互流程

**登录与 Token 管理**
```
启动 App → AppNavHost.kt 检查 TokenStorage.isAuthenticated()
    ├─ 未登录 → LoginScreen
    │       └─ 登录成功 → TokenStorage.saveTokens(access, refresh)
    │
    └─ 已登录 → MainScaffold（底部导航：工程 / 统计 / AI / 我的）
         └─ 发起请求 → AuthInterceptor 添加 Authorization: Bearer <token>
              ├─ 200 → 正常返回
              └─ 401 → 调用 POST /v1/auth/refresh 获取新 token
                    ├─ 成功 → TokenStorage.saveTokens(newAccess, newRefresh) → 重试原请求
                    └─ 失败 → TokenStorage.clearTokens() → 自动跳转 LoginScreen
```

## 🗃️ 数据库（PostgreSQL）

### 核心表结构

| 表名 | 用途 | 关键字段 |
|-----|-----|---------|
| **users** | 用户 | `id`, `username`, `nickname`, `password_hash`, `role`, `phone`, `created_at` |
| **projects** | 工程（核心） | `id`, `name`(客户地址), `description`, `status`(preparing/constructing/completed/canceled), `total_amount`, `created_by`, `created_at` |
| **subprojects** | 子项目 | `id`, `project_id`, `space_type_id`, `construction_plan_id`, `length`, `width`, `quantity`, `amount`, `created_by`, `created_at` |
| **project_workers** | 工程施工人员关联 | `project_id`, `user_id`, `work_days` |
| **project_user_status** | 用户结算状态 | `project_id`, `user_id`, `settlement_status`(unsettled/settling/settled) |
| **files** | 工程附件 | `id`, `project_id`, `filename`, `file_path`, `size`, `uploaded_by` |
| **project_history** | 工程修改历史 | `id`, `project_id`, `action`, `description`, `performed_by`, `created_at` |
| **space_types** | 空间类型字典 | `id`, `name` |
| **construction_plans** | 施工方案字典 | `id`, `name`, `unit`, `price` |
| **advances** | 预支记录 | `id`, `user_id`, `amount`, `date`, `remark` |
| **wage_settlements** | 工资结算单 | `id`, `user_id`, `project_id`, `total_amount`, `advance_deducted`, `net_amount`, `settled_at` |
| **messages** | 站内消息 | `id`, `user_id`, `title`, `content`, `type`, `is_read`, `created_at` |

### 数据库查询硬约束

| 约束 | 说明 | 实现位置 |
|-----|-----|---------|
| **时区处理** | 所有 `WHERE TO_CHAR(created_at, 'YYYY-MM')` 必须改为 `TO_CHAR(created_at AT TIME ZONE 'Asia/Shanghai', 'YYYY-MM')` | projectRepo.js countQuery + listQuery |
| **金额计算事务** | 子项目 `amount` 必须在事务内计算后直接写入，禁止在事务外计算后再 update | projectRepo.js createProjectWithSubproject |
| **参数化查询** | 所有 SQL 必须使用 `$1, $2, $3...` 参数占位，禁止字符串拼接 | 所有 repository 文件 |
| **工程总额重算** | 子项目增删改后必须调用 `recalculateProjectTotal(projectId)` | projectService.js |

### Token 生命周期管理

| 事件 | Token 状态 | 处理 |
|-----|-----------|-----|
| 登录成功 | 生成 (access, refresh) | 存入 Redis `refresh_token:{userId}`，返回给前端 |
| 正常请求 | access_token 有效 | 直接放行 |
| access_token 过期 | 返回 401 | Android 端 AuthInterceptor 自动调用 /refresh |
| 刷新成功 | 生成新的 (access, refresh) | Redis 更新旧 token，前端替换 |
| refresh_token 也过期 | 刷新失败 | 前端清除 token，跳回登录页 |
| 主动登出 | 调用 /logout | Redis 删除 refresh_token，前端清除 |
