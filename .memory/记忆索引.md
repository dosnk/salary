## 🗂️ 记忆文件清单与适用场景

### 01 - 项目核心记忆
- **文件**: `01-项目核心记忆.md`
- **何时读取**: 每次对话**必读**
- **内容概要**:
  - 项目名称与定位（三人行吊顶管理系统）
  - 业务领域：吊顶工程管理、工资结算、预支管理
  - 核心角色与权限体系（admin / constructor / documenter）
  - 核心业务流程与关键概念（工程、子项目、结算、预支）
  - 当前版本号与各端版本信息
  - 已实现功能清单

### 02 - 技术栈与架构
- **文件**: `02-技术栈与架构.md`
- **何时读取**: 涉及代码编写、架构调整、技术选型时
- **内容概要**:
  - 后端：Node.js + Koa + PostgreSQL + Redis + BullMQ
  - Web 前端：Vue 3 + Vite + TypeScript + Pinia
  - Android 端：Kotlin + Jetpack Compose + Hilt + Retrofit
  - 数据库表结构与关联关系
  - 代码分层架构（controller/service/repository）
  - API 接口规范与路由设计
  - AI 功能模块架构（大模型接入、排料引擎、知识库）

### 03 - 开发规范与约束
- **文件**: `03-开发规范与约束.md`
- **何时读取**: 编写代码、修复 bug、调整配置前**必读**
- **内容概要**:
  - 语言一致性：所有回复与代码注释使用中文
  - 金额计算：所有金额由后端计算，前端仅展示
  - 时区处理：使用 Asia/Shanghai，数据库存储带时区信息
  - 安全约束：SQL 参数化、JWT Token 规则、密码策略
  - 数据一致性：工程状态与结算状态的分离机制
  - UI 规范：主色调、间距、圆角、字体规范
  - 构建验证：修改后必须执行 `./gradlew assembleDebug`
  - 代码引用格式：所有文件引用使用 file:// 绝对路径

### 04 - 关键文件与快速定位
- **文件**: `04-关键文件与快速定位.md`
- **何时读取**: 需要快速定位代码位置时
- **内容概要**:
  - 后端关键入口与配置文件
  - 前端关键路由与组件映射
  - Android 端核心文件路径
  - 数据库脚本位置
  - 环境变量配置文件位置

### 05 - 历史会话摘要
- **文件**: `05-历史会话摘要.md`
- **何时读取**: 了解近期工作成果、避免重复任务、延续之前的开发方向时
- **内容概要**:
  - 每次重要对话的目标、决策、产出
  - 已修改的文件列表与修改摘要
  - 已发现但尚未修复的问题
  - 版本号变更记录
  - 待办任务跟踪

## ⚠️ 关键硬约束（所有任务均需遵守）

以下规则来自 [03-开发规范与约束.md](file:///f:/AIPoject/salary/.memory/03-开发规范与约束.md)，无论任务类型如何，均需遵守：

1. **中文回复**：所有对话回复使用中文
2. **中文注释**：代码注释使用中文
3. **金额由后端计算**：前端不得参与金额计算
4. **file:// 绝对路径引用**：代码中引用的文件路径需使用 `file:///f:/AIPoject/salary/...` 格式
5. **构建验证**：Android 端代码修改后必须执行 `.\gradlew assembleDebug` 验证通过
6. **权限体系**：admin 只读 + 系统配置，constructor 施工人员操作，documenter 只读
7. **时区**：Asia/Shanghai，数据库查询需带上时区处理
8. **修复不改原程序功能**：bug 修复不应引入新功能或改变原有逻辑
9. **Kotlin DTO 可空字段**：数据库允许 null 的字段，DTO 中应定义为可空类型
10. **Redis 缓存**：工程列表、用户信息等使用缓存，清除缓存需触发相应失效
