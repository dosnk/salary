# 三人行吊顶管理系统 - 业务流程文档 V4.0

## 0. 用户权限说明 V2.0

### 0.1 角色定义

| 角色 | 代码 | 定位 |
|------|------|------|
| 管理员 | admin | 只读监管 + 系统配置，**不干预业务操作** |
| 施工员 | constructor | 对自己参与的工程有完全操作权 |
| 资料员 | documenter | 查看全部工程(只读) + 新建工程 |

### 0.2 权限矩阵（最终版）

| 权限 | admin | constructor | documenter |
|------|:-----:|:-----------:|:----------:|
| 用户管理 | 全部 | 仅自己 | 仅自己 |
| 工程查看 | 全部 | 自己参与的 | 全部(只读) |
| 工程创建 | ✓ | ✓ | ✓ |
| 工程修改 | ✗ | 自己参与的 | ✗ |
| 工程删除 | ✗ | 自己参与的 | ✗ |

| 预支管理(增删改) | ✗(只能查看) | 自己 | ✗ |
| 统计查看 | ✓ | ✓(自己的) | ✗ |

### 0.3 施工员权限判断逻辑

施工员的权限范围是"自己参与的工程"，具体判断逻辑：

```
判断条件:
├── 工程创建者是自己 (projects.created_by = currentUserId)
└── 或者 自己是工程的施工人员 (project_workers.user_id = currentUserId)

SQL示例:
WHERE project_id IN (
  SELECT id FROM projects WHERE created_by = $1
  UNION
  SELECT project_id FROM project_workers WHERE user_id = $1
)
```

## 1. 核心业务流程

### 1.1 工程创建流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                         工程创建流程                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  1. 用户填写工程信息                                                 │
│     ├── 工程名称                                                     │
│     ├── 空间类型（从字典获取）                                        │
│     ├── 施工方案（从字典获取，含单价和单位）                           │
│     ├── 长度、宽度（厘米）                                           │
│     ├── 分配方式（平均分配/按工日分配）                               │
│     ├── 工程备注                                                     │
│     └── 施工人员列表                                                 │
│                                                                     │
│  2. 后端处理                                                         │
│     ├── 验证工程名称唯一性                                           │
│     ├── 转换空间类型名称为ID                                         │
│     ├── 转换施工方案名称为ID，获取单价和单位                          │
│     ├── 计算数量和金额（根据施工方案类型）                            │
│     ├── 创建工程记录                                                 │
│     ├── 创建子项目记录（含quantity和amount）                         │
│     ├── 更新工程总额                                                 │
│     └── 创建施工人员关联                                             │
│                                                                     │
│  3. 返回结果                                                         │
│     └── 工程创建成功，返回工程ID                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 施工方案计算公式

**重要：后端负责计算，前端只负责预览**

| 施工方案 | 计算公式 | 单价 | 单位 |
|---------|---------|------|------|
| 蜂窝平面 | 长 × 宽 × 单价 | 40元 | 面积(m²) |
| 半吊 | (长 + 宽) × 2 × 单价 | 60元 | 长度(m) |
| 二级平面 | 长 × 宽 × 单价 | 60元 | 面积(m²) |
| 铝扣平面 | 长 × 宽 × 单价 | 25元 | 面积(m²) |
| 窗帘盒 | 长 × 单价 | 20元 | 长度(m) |
| 发光走边 | (长 + 宽) × 2 × 单价 | 30元 | 长度(m) |
| 水坑 | (长 + 宽) × 2 × 单价 | 35元 | 长度(m) |
| 工程板 | 长 × 宽 × 单价 | 20元 | 面积(m²) |
| 其它方案 | 自定义计算 | 自定义 | 自定义 |

### 1.3 结算流程 V2.0（简化版）

```
旧流程(废弃): 选择工程 → 预览计算 → 提交结算 → 管理员审批 → 确认完成
新流程:       选择工程 → 预览计算 → 提交结算 → 施工员确认完成
```

- **不再需要管理员审批**
- 施工员对自己的工程自主完成结算确认
- 施工员只能结算自己参与且已完工(completed)的工程

---

### 1.4 预支流程

```
1. 施工员创建预支记录
2. 管理员可查看所有预支，但不能修改/删除
3. 结算时自动扣除未结算的预支金额
4. 结算完成后预支标记为已结算
```

---

### 1.5 统一计算服务

**重要：所有业务计算逻辑集中在后端 `services/calculation.js`**

导出函数：
- `calculateQuantity(unit, length, width)` - 根据单位类型计算数量
- `calculateSubprojectAmount(unit, length, width, price)` - 计算子项目金额
- `calculateUserWage(...)` - 计算用户工资分配
- `calculateAdvanceTotal(advanceIds)` - 计算预支总额
- `calculateFinalAmount(grandTotal, advanceTotal)` - 计算最终金额
- `calculateSettlementPreview(projectIds, currentUserId)` - 计算结算预览

## 2. AI业务流程（新增）

### 2.1 AI对话流程

```
用户提问 → 意图路由
              │
    ┌─────────┼─────────┐
    ▼         ▼         ▼
 排料计算   数据查询   知识问答
    │         │         │
    ▼         ▼         ▼
 本地引擎  Function    RAG检索
 (ms级别)   Calling    + 大模型
    │         │         │
    │         ▼         │
    │    权限过滤查询    │
    │    (admin看全部   │
    │     constructor看 │
    │     自己参与的)   │
    │         │         │
    └─────────┼─────────┘
              ▼
        大模型综合回复
```

### 2.2 排料计算流程

```
用户输入空间尺寸 + 选择材料
        │
        ▼
从知识库(material_params)加载材料参数
        │
        ▼
本地排料引擎计算(纯本地，<50ms):
├── 面材排料: 计算完整面板 + 裁切面板 + 损耗
├── 龙骨计算: 主龙骨 + 副龙骨 + 边龙骨
├── 收边条计算: 按周长和标准长度
└── 配件计算: 吊件/膨胀螺丝/连接件
        │
        ▼
生成SVG渲染数据(后端生成)
        │
        ▼
(可选) 发送排料结果给大模型，获取优化建议
        │
        ▼
返回: { 排料结果, SVG数据, AI建议, 费用估算 }
```

### 2.3 知识库RAG流程

```
用户提问: "蜂窝板施工规范是什么？"
        │
        ▼
1. 向量嵌入(将问题转为向量)
        │
        ▼
2. pgvector相似度搜索(在ai_knowledge_chunks表)
   → 找到相关文档片段
        │
        ▼
3. Context组装(相关片段 + 用户问题 + 系统提示)
        │
        ▼
4. 发送给大模型生成回答
        │
        ▼
5. 返回带来源引用的回答
```
